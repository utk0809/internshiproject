# Project Handover Report: SQL Injection Educational Demo

## 1. Overview
This project is an interactive, educational web application designed to demonstrate **SQL Injection (SQLi) vulnerabilities** and their corresponding **Secure Coding solutions** (Parameterized Queries). It is built using HTML, CSS, and PHP, making it lightweight and easy to deploy on any standard web server.

## 2. Key Components & Features

### A. Educational Home Page (`index.php`)
- **Visual Attack Masterclass:** A custom-built, animated flow diagram that explains the SQLi attack lifecycle step-by-step (Input -> Concatenation -> Logic Change -> Bypass).
- **Architectural Diagram:** An AI-generated visual representation of an attack scenario (`sqli_diagram_1773407391766.png`).
- **Interactive UI:** Features a modern, dark-themed "glassmorphism" design with glowing borders and subtle micro-animations.

### B. Vulnerable Simulator (`login.php`)
- **Intentional Vulnerability:** Uses raw string concatenation for SQL queries to demonstrate how `' OR 1=1 --` bypasses authentication.
- **Attack Simulation Lab:** A compact, one-line grid of payloads that allows users to "pre-fill" the login form with common SQLi strings without auto-submitting. This encourages the user to read and understand the payload before executing the attack.

### C. Secure Version (`secure.php`)
- **Defensive Implementation:** Uses simulated **Parameterized Queries** (PDO style) where user input is treated as pure data, never as executable code.
- **Comparative Lab:** Includes the same Attack Lab as the vulnerable page, allowing users to see exactly how the secure version neutralizes every injection attempt.
- **Deep-Dive Explanations:** Detailed sections explaining Prepared Statements, Input Validation, and Query Compilation.

### D. Global Styling (`style.css`)
- **Unified Design System:** All components, including the attack labs, interactive boxes, and animated flows, are styled in a single, well-organized CSS file.
- **Responsive Layout:** The design adapts to different screen sizes, ensuring the diagrams and tables remain legible on mobile and desktop.
